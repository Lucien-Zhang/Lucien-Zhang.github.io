Git + Hexo 部署文章时出错排故方法

Feb 27, 2018 | Technology | 阅读 | 709 字 | 3 分钟

两个多月没写文章，这几天写了一篇文章居然发现没有部署上去！？原来是Git软件版本问题。

这是直接使用 hexo d时出现的错误代码

排查

使用hexo clean、hexo generate命令没有问题

使用ssh -T git@github.com命令测试SSH Key，没有问题

登陆 Github - Setting - SSH and GPG keys 发现SSH最后使用的时间停留在17年12月份

.js、Git两个软件、Blog的文件夹都完好

分析:

  1. 怀疑是Win10这两个月间的一次系统更新把某个软件或者注册表打成内伤了
  2. Hexo 或者Git for Windows 版本过低
  3. 宽带运营商问题，先前是上海电信，回来是广东移动（广东移动没办法登陆steam客户端打开网页显示无服务错误而上海电信不会）

排故

  1. 卸载Git(V2.13),安装新版本Git(V2.16)，检查SSH连接情况:
    git config --global user.name #检查Github用户名 通过
    git config --global user.email #检查Github注册邮箱 通过
    ssh-keygen -t rsa -c #生成SSH密匙 通过
    ssh -T git@github.com #检查SSH连接情况 通过（SSH最后使用时间更新为“最近一周内使用过”且文字为绿色）
    可以正常使用

    宽带运营商问题应该算解决了吧，因为更新了最新的Git版本，使用上海电信后也能正常部署文章。

    总结

    很幸运的一次就解决了问题，运气真好。这段代码看得懂，但是看不明白，还需要多学习网络通讯协议和Git命令。
